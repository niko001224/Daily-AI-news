## 📌 项目简介

本仓库用于**自动化爬取、存储与整理 AI 行业资讯及国内云厂商动态**，所有内容以 **Markdown** 格式进行结构化沉淀，构建一个长期可积累、可分析、可回溯的行业资讯知识库。

**系统机制说明：**
- ⏰ 每日 **09:00** 定时任务
- 🔍 通过Wide Research获取 **过去 24 小时内** 的最新资讯
- 📥 覆盖内容包括：
  - AI 行业新闻、技术进展、模型发布、产品动态
  - 国内云厂商相关动态（云服务、算力平台、AI 平台、行业解决方案等）

## 📁 目录结构说明

- **每个文件夹对应一周的资讯数据**
- 文件夹内按来源/主题/时间存放当周爬取的 Markdown 文档：
  - AI 行业资讯
  - 国内云厂商资讯
- 按时间维度归档，支持历史回溯、趋势分析与结构化整理

## 📊 周报生成机制

- 每周自动读取对应时间文件夹内的所有 Markdown 文档
- 对当周 AI 行业动态与云厂商资讯进行整理、聚合与归类
- 生成一份 **周度综合资讯报告**
- 统一存储至报告文档中，作为阶段性总结与趋势参考

## 🎯 项目目标

- 构建一个**自动更新的 AI + 云计算行业情报系统**
- 实现 **资讯获取 → 结构化存储 → 周期性总结** 的闭环
- 为行业研究、技术调研、产品分析与战略判断提供数据基础支撑

## 🧾 命名规范说明

### 📅 周级目录
- 命名规则：`YYYY_w_MMDD`
- 示例：`2026_w_0126`
- 含义：**以该周周一的日期作为周标识**
- 用途：聚合该自然周内的所有日级资讯数据

### 🧠 AI 资讯文件
- 路径：`data/YYYY_w_MMDD/ai/`
- 命名规则：`ai_YYYY_MM_DD.md`
- 示例：`ai_2026_01_22.md`
- 内容：该日期 **过去 24h 内** 的 AI 行业资讯

### ☁️ 云厂商资讯文件
- 路径：`data/YYYY_w_MMDD/cloud/`
- 命名规则：`cloud_YYYY_MM_DD.md`
- 示例：`cloud_2026_01_22.md`
- 内容：该日期 **过去 24h 内** 的国内云厂商动态

### 📘 周级 README.md
- 用于说明：
  - 该周时间范围（周一 ~ 周日）
  - 数据是否完整
  - 抓取异常或缺失说明

## 🔄 数据生成与汇总节奏

- ⏰ **每日 09:00**
  - 生成：
    - `ai/ai_YYYY_MM_DD.md`
    - `cloud/cloud_YYYY_MM_DD.md`
- 📊 **每周**
  - 汇总对应 `YYYY_w_MMDD` 目录下的所有日级数据
  - 生成周度综合报告
- 📝 周报统一存放于 `reports/` 目录
